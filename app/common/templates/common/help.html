    Documentation & Reference

    Complete guide to Traefik Manager — concepts, fields, workflows, and troubleshooting

    Traefik Manager stores all configuration in etcd using a key-value store. Traefik reads this configuration dynamically — changes take effect within seconds, no restart needed.

    HTTP Layer
    Routers → match incoming requests
    Services → forward to backend URLs
    Middlewares → transform requests
    ServersTransport → backend TLS/conn config
    TCP / UDP Layer
    TCP Routers → match by SNI/HostSNI
    TCP Services → backend server pools
    TCP Middlewares → IP allow-list / rate
    UDP Routers + Services → raw UDP
    TLS Layer
    TLS Options → cipher suites, versions
    TLS Stores → default certificates
    Domains → ACME resolvers per domain
     

    etcd Key Prefix: traefik/

    Example keys: traefik/http/routers/my-router/rule, traefik/tcp/services/my-svc/loadBalancer/servers/0/address

    1. Create a Service — define the backend URL (e.g. http://app:8080)
    2. Create Middleware(s) (optional) — add auth, rate limiting, prefix stripping
    3. Create a Router — set the matching rule, link to service + middlewares, configure TLS
    4. Verify — check the Routers list; the TLS badge should appear if enabled
    5. Test — make an HTTP/HTTPS request to the domain; Traefik picks up changes within 2–5 seconds
    Tip: Always create the Service before the Router — the Router form shows a dropdown of existing services.

    Services define the backend servers that handle requests. Each service maps to one backend URL with load-balancing support.

    FieldRequiredDescription
    Service Name✅Unique identifier referenced by routers
    Backend URL✅Full URL including scheme and port, e.g. http://backend:8080
    The Services list shows a Backend URL column with clickable links. Use Edit to safely update the backend — changing a service name requires deleting and recreating the router that references it.

    UDP routing is stateless — no TLS, no SNI matching. Routers simply forward all traffic on a given entrypoint to a service.

    EntityKey Fields
    UDP RouterService (required), Entrypoints (e.g. udp)
    UDP ServiceServer addresses: host:port, comma-separated for multiple
    Delete Protection: Deleting a UDP Service that is referenced by a UDP router is blocked.

    Export

    Downloads a JSON snapshot of all HTTP routers and services currently stored in etcd. Use for backup or migration.

    Traefik Manager can connect to multiple etcd instances. Connections are stored in a local SQLite database and persist across restarts.

    1. Navigate to Config → etcd Connections in the sidebar
    2. Click Add Connection — enter a name, the etcd URL (e.g. http://10.0.1.5:2379), and an optional description
    3. Click Connect on any saved connection to switch the active endpoint immediately
    4. The sidebar bottom always shows the current connection name and health status
    5. On restart, the last active connection is automatically used
    The ETCD_URL environment variable sets the initial default connection when no DB exists. Once a connection is saved in the DB, the DB value takes precedence.

    • Default credentials: admin / admin
    • On first login, you are required to change the password before accessing any other page
    • Passwords are stored as salted SHA-256 hashes in a local SQLite database (traefik_manager.db)
    • To change your password at any time, click the key icon next to your username in the sidebar
    • To sign out, click the icon in the sidebar
